Solution Guide

⚡ PowerApps Knowledge Base Chat Bot

A low-code chat interface built with Power Apps + Power Automate + SharePoint that lets users ask questions and get instant answers from your organisation's knowledge base.

★ Architecture Overview

💬
Frontend

Power Apps

Canvas app embedded in SharePoint. Provides the chat UI: message bubbles, text input, and send button.

⚡
Logic / Brain

Power Automate

Triggered by the app. Searches SharePoint for keyword matches and returns the best answer.

📋
Knowledge Base

SharePoint List

Stores Q&A pairs. Each row has keywords and a corresponding answer. No code required to update.

ℹ️
Prerequisites You need a Microsoft 365 account with access to Power Apps, Power Automate, and SharePoint Online. No premium connectors are required — this solution uses only standard connectors.

1 Create the SharePoint Knowledge Base List

1

Go to your SharePoint site and click New → List → Blank list.

2

Name the list exactly: ChatKnowledgeBase. Click Create.

3

Add a Multiple lines of text column named Answer.

4

Add a Choice column named Category with choices: General, HR, IT, Finance, Operations, Other.

5

Add a Number column named Priority (default value: 1).

6

Add a Yes/No column named IsActive (default: Yes).

7

Populate the list using sharepoint-list-schema.json as a reference. The Title column holds comma-separated keywords.

✅
Tip – Writing good keywords Separate keywords with commas: password reset, forgot password, login help. The flow checks if any keyword is contained within the user's question. Avoid short common words like "the" or "is".

2 Build the Power Automate Flow

1

Go to flow.microsoft.com → Create → Instant cloud flow.

2

Name it ChatBot-KnowledgeBase-Search. Select trigger: Power Apps (V2). Click Create.

3

In the trigger, add an input: Name = UserQuestion, Type = Text.

4

Add action: Initialize variable → name varBestAnswer, type String, value = your fallback message (e.g. "I couldn't find an answer, please contact support.").

5

Add action: Initialize variable → name varMatchFound, type Boolean, value false.

6

Add action: Compose → expression: toLower(triggerBody()['UserQuestion']). This normalises the question to lowercase.

7

Add action: SharePoint – Get items. Set Site to your SharePoint URL, List to ChatKnowledgeBase, Filter Query to IsActive eq 1, Order By to Priority asc, Top Count to 100.

8

Add action: Apply to each over the Get Items output. Inside the loop, add a Condition: varMatchFound equals false.

9

Inside the "Yes" branch, add another Apply to each over split(toLower(items()?['Title']), ',').

10

Inside the inner loop, add a Condition: contains(outputs('Compose'), trim(item())) AND varMatchFound equals false.

11

In the "Yes" branch: Set variable varBestAnswer to items()?['Answer'], then Set variable varMatchFound to true.

12

After the outer loop, add: Respond to a PowerApp or flow. Add two outputs: answer (Text) = varBestAnswer; matchFound (Boolean) = varMatchFound.

13

Save and run a test with a sample question to verify the response.

⚠️
Important – Concurrency settings Do not enable concurrency on the "Apply to each" loops. Parallel execution causes race conditions when reading/writing the varMatchFound variable.

3 Build the Power Apps Canvas App

1

Go to make.powerapps.com → Create → Blank app → Blank canvas app. Name it KnowledgeBaseChatBot.

2

Add a Rectangle at the top (height 70 px, fill #0078D4) as the header. Add a white bold Label on top: "Knowledge Base Assistant".

3

Add a Vertical Gallery below the header. Set Items to colMessages. Enable auto-grow template height.

4

Inside the gallery template, add two Rectangles (bot bubbles on left, user bubbles on right) and a Label for message text. Use ThisItem.role to control visibility and colour.

5

Add a Text Input at the bottom for the user's question. HintText: "Type your question here...".

6

Add a Send Button next to the input with the OnSelect formula below.

7

Set the App OnStart formula below.

8

In the left sidebar, click the ⚡ Power Automate icon → Add flow → select ChatBot-KnowledgeBase-Search.

9

Save and Preview (F5) to test the app.

App OnStart Formula

// Initialise an empty messages collection with a welcome message
Clear(colMessages);
Set(varIsLoading, false);
Collect(
    colMessages,
    {
        role: "bot",
        message: "Hello! I'm your Knowledge Base Assistant. Ask me anything!",
        timestamp: Now()
    }
)

Send Button OnSelect Formula

// Guard: skip if empty or already loading
If(IsBlank(Trim(txtUserInput.Text)) || varIsLoading, Return);

// Capture and clear the input
Set(varCurrentQuestion, Trim(txtUserInput.Text));
Reset(txtUserInput);

// Add user message to chat
Collect(colMessages, {
    role: "user",
    message: varCurrentQuestion,
    timestamp: Now()
});

// Show loading indicator
Set(varIsLoading, true);

// Call the Power Automate flow
Set(
    varFlowResponse,
    ChatBot_KnowledgeBase_Search.Run(varCurrentQuestion)
);

// Add bot response to chat
Collect(colMessages, {
    role: "bot",
    message: varFlowResponse.answer,
    timestamp: Now()
});

// Hide loading indicator
Set(varIsLoading, false)

4 Embed the App in SharePoint

1

In Power Apps Studio, click File → Save → Publish.

2

Click Share and share the app with your organisation or specific users/groups.

3

On your SharePoint page, click Edit → + Add a new web part.

4

Search for and select the Power Apps web part.

5

Paste your app's URL or search for it by name. Click Add.

6

Resize the web part and click Republish on the SharePoint page.

5 Maintaining the Knowledge Base

No code changes needed to update Q&A content. Simply edit the SharePoint list:

ActionWhat to do
Add a new Q&AAdd a new row to ChatKnowledgeBase. Fill in the keywords (Title) and the Answer.
Disable an answerSet IsActive to No — the bot skips it without deleting your work.
Change prioritySet a lower Priority number to make that answer match first when multiple entries could match.
Refine keywordsEdit the Title field to add or remove comma-separated keywords.
✅
Best practices Keep keywords specific. Use 3–5 specific terms per entry and test regularly by asking the bot questions a real user might ask. Use the Priority column to control which answer wins when multiple entries match.
